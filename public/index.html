Git Permission Layer For AI Agents

Five agents, five keys, one repo, zero risk.

repo.box is a local tool that sits in the git signing path. Agents run normal git commands. Every commit, merge, and workflow transition is checked against .repobox-config before it lands.

The Problem

GitHub permissions were built for humans, not autonomous loops.

AI agents do not stay inside the assumptions of branch protection and org roles. They open branches, rewrite files, spawn new tasks, and call external services. Git still sees a generic user. That is too coarse.

Hosted permissions are role-based

An agent can inherit a human seat, then act across the whole repo. No file scope. No per-request auth. Weak attribution.

Bots self-escalate by accident

One mistaken merge, one secret touched, one workflow bypassed. The blast radius is the repository, not the task.

Audit trails are shallow

You know a token pushed. You do not know which key approved which transition and under which policy snapshot.

Agents need repo-local policy

Permissions should travel with code, review like code, and fail closed like code.

How It Works

Four primitives. Everything else composes.

1. Identity

Use an EVM wallet as the git signing identity. Commits are cryptographically signed by the agent key, not by a shared bot account.

2. Permissions

Store branch and file rules in .repobox-config. repo.box is the signing program, so authorization happens inside normal git flows.

3. Groups

Resolve “is address X in group Y?” from static lists, onchain token gates, or external APIs. Policy does not care where membership comes from.

4. Workflows

PRs, approvals, governance, release trains. They are all state machines attached to repo events and enforced by the same key material.

Full Config

A realistic .repobox-config

This is the point. Policy lives in the repo, versioned with the code it protects. Branches, files, groups, workflow transitions, and external auth all share one source of truth.

chmod-style rules for git, not the filesystem
version: 1

identity:
  format: evm
  requireSignedCommits: true

groups:
  core:
    resolver: static
    members:
      - "0xFRAAAAAA00000000000000000000000000000001"
      - "0xOCEAAAA000000000000000000000000000000002"

  agents.lint:
    resolver: static
    members:
      - "0xA11CE00000000000000000000000000000000003"
      - "0xB0B00000000000000000000000000000000000004"

  reviewers:
    resolver: onchain
    chainId: 8453
    contract: "0x8128000000000000000000000000000000000812"
    rule:
      type: erc721-balance
      minBalance: 1

  governance:
    resolver: http
    method: POST
    url: https://policy.repo.box/v1/group-membership
    auth: erc-8128

permissions:
  - group: core
    allow:
      branches: ["main", "release/*", "agent/*"]
      paths: ["**"]
      actions: ["commit", "merge", "workflow.transition"]

  - group: agents.lint
    allow:
      branches: ["agent/*"]
      paths: ["src/**", "package.json", "pnpm-lock.yaml"]
      actions: ["commit"]
    deny:
      paths: [".env*", "secrets/**", ".github/workflows/release.yml"]

  - group: reviewers
    allow:
      branches: ["main"]
      paths: ["src/**", "docs/**"]
      actions: ["workflow.transition:review.approve"]

workflows:
  review:
    onBranch: "main"
    initial: draft
    states:
      draft:
        transitions:
          submit:
            to: in_review
            by: core

      in_review:
        transitions:
          approve:
            to: approved
            by: reviewers
            quorum: 2
          reject:
            to: changes_requested
            by: reviewers

      changes_requested:
        transitions:
          resubmit:
            to: in_review
            by: core

      approved:
        transitions:
          merge:
            to: merged
            by: core
            requires:
              branchClean: true
              signedHead: true

branchPolicies:
  "agent/*":
    ephemeral: true
    inherit: false
    createdBy: core
    allowedGroups: ["agents.lint", "core"]
    mergeTarget: main
    deleteOnMerge: true

httpAuth:
  default:
    scheme: erc-8128
    domain: repo.box
    statement: "Signed request from repo.box policy engine"

Branch Sandboxing

Feature branches are agent sandboxes.

Give an agent a branch. Give it a key. Give it the smallest possible path scope. When the branch merges or gets deleted, the permission surface disappears with it. No standing privileges. No self-escalation.

Agent keys cannot mutate main directly unless policy says so.
An agent can be allowed to write src/** on agent/* and still be blocked from .github/ and secrets.
Merging is a separate permission and can be bound to workflow state plus reviewer quorum.
sandbox flow
main
  |
  +-- agent/ocean/fix-copy
  |      key: 0xA11CE...
  |      paths: docs/**, src/ui/**
  |      actions: commit
  |      merge: denied
  |
  +-- agent/bob/run-lint
         key: 0xB0B...
         paths: src/**, package.json
         actions: commit
         merge: denied

approved review on main
  -> merge executed by core key
  -> ephemeral branch deleted
  -> agent permissions vanish with branch

Built On Standards

No new account system. No hidden control plane.

ERC-8128 auth: signed HTTP requests instead of long-lived JWTs or API keys. Every request stands on its own signature.

EVM signatures: one identity works for commits, policy checks, and external resolvers.

git-native shim: agents still use git add, git commit, git merge. repo.box sits under normal commands instead of inventing a new VCS.

Get Started

Install the shim. Add policy. Keep using git.

repo.box is for repositories that want AI contribution without turning the repo into a shared root shell. Setup should feel closer to prettier or eslint than a platform migration. 

# install
curl -fsSL https://repo.box/install.sh | bash

# turn repo.box into the signing path
git config gpg.program repobox
git config commit.gpgsign true

# bind this repo to an agent key
git config user.signingkey evm:0xA11CE00000000000000000000000000000000003

# add repo policy
touch .repobox-config
git add .repobox-config
git commit -m "policy: add repo.box guardrails"

# agents keep using plain git
git checkout -b agent/ocean/fix-lint
git commit -am "lint: cleanup"
