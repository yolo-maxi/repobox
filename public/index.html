repo.box

Git permission layer that makes repositories safe for AI agents.

EVM keys, a git shim, and a .repobox.yml file that stands between your agents and your codebase. One config file. Per-agent permissions. Enforced before anything touches your repo.

↓ scroll

What repo.box Does

Permission Layer
git shim
Intercepts git commands. Every commit, merge, push is checked against .repobox.yml before it happens.
EVM Identities
crypto
Agents sign commits with secp256k1 keys. No SSH keys, no GPG complexity. Just wallet addresses.
Sandbox by Default
security
Agents get feature branches. They can't touch main, can't edit the config. Structurally impossible to escalate.

Use Cases

Solo + Codex
simple
One developer, one AI agent. Agent gets feature branches, can't touch main or the config.
Before: Manual review of every AI commit
After: Auto-sandboxed, merge when ready
Team + Agents
standard
Founders control everything. Multiple agents work in parallel on feature branches.
Before: Shared credentials, trust issues
After: Per-agent keys, audit trail
Multi-Agent Swarm
advanced
Orchestrator agent spawns sub-agents. Each gets scoped permissions via append-only config changes.
Before: Can't safely delegate
After: Hierarchical trust model

Design Principles

  • →Agents use normal git — the shim is invisible
  • →EVM keypairs as the identity layer
  • →Every agent signs commits with its own key
  • →Everything lives in the repo — permissions, rules, workflows, one place
  • →Enforced locally, no server required
